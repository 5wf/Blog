<p>In <a href="{{BASE_PATH}}/2008/06/03/howtocode-youread-community-getriebene-professionelle-applikationsentwicklung/">dem letzten "ReadYou" Post</a> ging es allgemein um den Gesamtplan - als erstes möchte ich nochmal komplett ohne Code arbeiten und auch keine großen Gedanken zur Architektur machen. </p> <p>Hier geht es um die <a href="http://de.wikipedia.org/wiki/Anforderung_%28Informatik%29">Anforderungen</a> die ich an das System habe. Neben den bereits gestellten Qualitätsanspruchen (Source Control, Tests, Dokumentation) sollten auch gewisse Grundfunktionen enthalten sein. </p> <p><strong>Wichtiger Hinweis:</strong> Ich werde die Anforderungen bewusst "simpel" fassen - im professionellen Umfeld sollte an dieser Stelle besonders viel mit dem Kunden geredet werden um herauszufinden, was er eigentlich will!</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image422.png"><img style="border-right: 0px; border-top: 0px; border-left: 0px; border-bottom: 0px" height="616" alt="image" src="{{BASE_PATH}}/assets/wp-images/image-thumb401.png" width="483" border="0"></a> </p> <p><a href="{{BASE_PATH}}/assets/wp-images/image423.png"><img style="border-right: 0px; border-top: 0px; border-left: 0px; border-bottom: 0px" height="633" alt="image" src="{{BASE_PATH}}/assets/wp-images/image-thumb402.png" width="340" border="0"></a> </p> <p><strong>Was genau muss die Software/Plattform leisten?</strong></p> <p>Bilder sagen manchmal mehr als tausend Worte (und vor allem empfinde ich es dann meist viel logischer, wie das hinterher aussehen muss):</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image424.png"><img style="border-right: 0px; border-top: 0px; border-left: 0px; border-bottom: 0px" height="324" alt="image" src="{{BASE_PATH}}/assets/wp-images/image-thumb403.png" width="537" border="0"></a> </p> <p>Aufmerksamen Lesern des Blogs sollte dieses Bild aus dem <a href="{{BASE_PATH}}/2008/04/22/ui-prototyping-mit-powerpoint-2007/">UI Prototyping mit Powerpoint</a> bekannt vorkommen.</p> <p>Ich liste jetzt mal alle Funktionen auf:</p> <ul> <li><strong>User-Bereich:</strong></li> <ul> <li>Login/Logout/Register/Password-Recovery</li> <li>Admin hat Management-Oberfläche</li> <li>User kann Profildaten speichern</li> <li>User sollte sich auch mit Open ID, Windows Live ID etc. anmelden können</li> <li>User kann andere User suchen etc.</li> <li>User kann Bücher anlegen (später evtl. auch Blogs)</li> <li>"Meine Seite" ähnlich wie StudiVZ Home</li> <li>Community - "Gruppen" etc. </li></ul> <li><strong>Bücher-Bereich:</strong></li> <ul> <li>Anlegen / Löschen / Managen</li> <li>Bewerten</li> <li>Taggen</li> <li>Kategorisierung der Bücher</li> <li>Amazon-Anbindung</li> <li>Rezessionen verfassen (oder man nimmt die Amazon-Rezessionen über die API ;) )</li></ul> <li><strong>Allgemeines:</strong></li> <ul> <li>Aktive User ähnlich wie in den ASP/MSDN Foren</li> <li>Neuste Bücher wie bei YouTube "Videos right watched now..."</li></ul></ul> <p><strong>Um es kurz zu sagen:</strong> <u>Typische Communityseite</u> - wobei ich die Anforderungen hier nicht so eng fasse.</p> <p><strong>Eine interessante Sachen die ich gerne einbauen würde:</strong></p> <ul> <li>Providermodell für Authentifizierung &amp; Userdaten</li></ul> <p>Das ASP.NET Membership-System gefällt mir nicht wirklich - es ist nett, hat aber seine Tücken und ist bei manchen Sachen (Profiles z.B.) nicht wirklich schön. <br>Zudem möchte ich die Authentifzierung von den eigentlichen Nutzerdatenzugriff trennen, weil ich bislang häufig bei Projekten folgendes Szenario hab:</p> <ul> <li>Oracle-DBs mit wilden Benutzerdaten</li> <li>AD / LDAP als Authentifizierung, allerdings stammen die wirklichen Userdaten dann aus einer anderen DB</li> <li>Profildaten etc. werden extra gespeichert</li></ul> <p>Insbesondere durch die Anforderung, die ich mir selbst gestellt habe, als Authentifizerung auch OpenID, Windows Live etc. zu unterstützen, wären die Userdaten und die Userauthentifizierung sowieso getrennt.</p> <p><strong>"AuthenticationRepository" &amp; "UserRepository":</strong></p> <p>Ich bin sehr angetan von <a href="http://blog.wekeroad.com/mvc-storefront/mvcstore-part-3/">Rob Conerys Architektur</a> mit den <a href="http://de.wikipedia.org/wiki/Pipes_and_Filters">Pipes &amp; Filters Modell</a> und dem <a href="http://msdn.microsoft.com/en-us/library/system.linq.iqueryable.aspx">IQueryable Interface</a>.</p> <p>Als momentane Idee steht daher solch eine "Grobe"-Architektur:</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image425.png"><img style="border-right: 0px; border-top: 0px; border-left: 0px; border-bottom: 0px" height="96" alt="image" src="{{BASE_PATH}}/assets/wp-images/image-thumb404.png" width="514" border="0"></a> </p> <p>Diese "Architekturgedanken" sind hier für die Erklärung der Anforderung zu verstehen. Code etc. wird es erst später geben ;)</p> <p><strong>Data:<br></strong>Hier versteh ich die verschiedenen Datenquellen - z.B. eine Oracle-DB, eine SQL-DB, Windows Live oder OpenID (wenn es um die Authentifizierung geht).</p> <p><strong>Repository;<br></strong>Hier ist die Datenzugriffsschicht - als wäre hier angenommen LINQ to SQL/ADO.NET EF/NHibernate/SubSonic oder die Windows Live API etc. einzubauen.</p> <p><strong>Service:<br></strong>Der Service sollte so lose wie möglich an das "Backend" gekoppelt sein - ich denke dies hier ist ein nettes Einsatzgebiet von <a href="http://de.wikipedia.org/wiki/Dependency_Injection">Dependency Injection</a>. Dieser Serivce wird im Frontent aufgerufen und gibt das an die entsprechenden Repositories weiter.</p> <p><strong>Mein Wunschtraum:</strong>&nbsp;</p> <p>Der Idealzustand dieser Plattform wäre, wenn ich an einer Codestelle oder über Konfiguration mein "Datenquellen" angeben kann, z.B.:</p> <p>Variante A)<br>Nimm als Authentifizierungsquelle und Userquelle den SQL Server.</p> <p>Variante B)<br>Nimm als Authentifizierungsquelle das Active-Directory und als Userquelle die Oracle DB </p> <p>Am Ende muss man "nur" einen entsprechenden Provider bereitstellen und ich will Service nichts großes ändern müssen - ob das so klappt, wage ich jetzt mal zu bezweifeln, da die Anmeldedaten auch unterschiedlich sind - aber das wird eine interessante Sache für die nächsten Blogposts ;)</p> <p><strong>Alles klar?<br></strong><strong>Kunde:</strong> "Falls Sie noch Fragen haben, melden Sie sich einfach. Ich denke, dass ist eine machbare kleine Anwendung - das dauert bestimmt nicht lange."</p> <p><strong>Programmierer:</strong></p> <p><a href="{{BASE_PATH}}/assets/wp-images/image426.png"><img style="border-right: 0px; border-top: 0px; border-left: 0px; border-bottom: 0px" height="316" alt="image" src="{{BASE_PATH}}/assets/wp-images/image-thumb405.png" width="360" border="0"></a> </p> <p><strong>Feedback:</strong><br>Auch wenn ich einen möglichst "professionellen" Ansatz verfolgen möchte, will ich hier möglichst bald euch Code präsentieren. Die Rubrik heisst ja auch "HowToCode" und nicht "HowToRequirementsManagement", daher sei mir hoffentlich die spärlichen Anforderungen verziehen ;) </p> <p>Aber wenn ihr noch Vorschläge habt (was unbedingt mit rein müsste, und was man mit bedenken sollte), dann immer her damit.</p> <p><strong>Ausblick:<br></strong>Die Idee mit der Authentifizierung/Userdata-Sache werde ich sicherlich erstmal prototyisch das nächste mal in Angriff nehmen und genauer auf die anderen Architekturpunkte eingehen.</p>
