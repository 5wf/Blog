<p>Willkommen im 100% Microsoft-approved Developer Tooling: Seit letzte Woche sind die <a href="https://tfs.visualstudio.com/">Team Foundation Services</a> in den RTM Status gegangen – d.h. man kann und darf damit produktiv arbeiten, allerdings gibt es noch keine absoluten Angaben zum Preis. </p> <p><strong>Nur soviel ist jetzt bekannt (ohne Gewähr) :</strong></p> <p>- Teams bis zu 5 Mann können kostenlos die Team Foundation Services nutzen</p> <p>- Keine Beschränkungen in Anzahl der Projekte</p> <p>- Build-Service wird vermutlich irgendwann in einem bestimmten Rahmen Geld kosten. Ist aber noch im Beta Status</p> <p>- Preise erst 2013, bis dahin ist es für alle kostenlos</p> <p>Mehr Details gibt es <a href="http://blogs.msdn.com/b/bharry/archive/2012/10/31/team-foundation-service-rtm.aspx">hier</a>.</p> <p><strong>Zum Team Foundation Service</strong></p> <p>Kurzfassung: Der Team Foundation Service ist ein gehosteter Team Foundation Server von Microsoft.</p> <p>Wer schon mit dem TFS zutun hatte, der kennt vermutlich Freud- und Leid damit. Allerdings komme ich nicht umhin einen grossen Respekt Microsoft auszusprechen, denn der Einstieg empfinde ich als äußerst gelungen und ich stelle mir den TFS auch nicht gerade als eine sehr simple Cloud Applikation vor ;)</p> <p><strong>Kurze Einstieg</strong></p> <p>Wir erstellen eine Standard MVC App:</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1618.png"><img title="image" style="border-top: 0px; verbinden ihn mit unserem Microsoft Account) und erstellen ein Teamproject. </p> <p>Nun können wir im Visual Studio gewohnt die Adresse eingeben:</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1619.png"><img title="image" style="border-top: 0px; border-right: 0px; border-bottom: 0px; border-left: 0px; display: inline" border="0" alt="image" src="{{BASE_PATH}}/assets/wp-images/image_thumb778.png" width="582" height="428"></a></p> <p>Und nach einer Anmeldung…</p> <p>&nbsp;</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1620.png"><img title="image" style="border-top: 0px; Hier sind mir ein paar Sachen noch aufgefallen – kann aber auch generell am TFS 2012 liegen:</p> <p>Der Output eines Builds liegt im Source Control – alles andere stell ich mir in einer Cloud Umgebung auch nicht so einfach vor.</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1623.png"><img title="image" style="border-top: 0px; border-right: 0px; border-bottom: 0px; border-left: 0px; display: inline" border="0" alt="image" src="{{BASE_PATH}}/assets/wp-images/image_thumb782.png" width="574" height="364"></a> </p> <p>Es gibt zudem ein Azure Deployment Process Template:</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1624.png"><img title="image" style="border-top: 0px; border-right: 0px; border-bottom: 0px; border-left: 0px; display: inline" border="0" alt="image" src="{{BASE_PATH}}/assets/wp-images/image_thumb783.png" width="594" height="176"></a> </p> <p>So wie es aussieht, kann man aber auch Build Templates verändern bzw. andere hochladen. </p> <p><strong>Web UI</strong></p> <p>Natürlich gibt es auch eine schicke Web UI</p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1625.png"><img title="image" style="border-top: 0px; border-right: 0px; border-bottom: 0px; border-left: 0px; display: inline" border="0" alt="image" src="{{BASE_PATH}}/assets/wp-images/image_thumb784.png" width="623" height="378"></a> </p> <p><a href="{{BASE_PATH}}/assets/wp-images/image1626.png"><img title="image" style="border-top: 0px; border-right: 0px; border-bottom: 0px; border-left: 0px; display: inline" border="0" alt="image" src="{{BASE_PATH}}/assets/wp-images/image_thumb785.png" width="639" height="441"></a> </p> <p></p> <p><strong>Hui oder Pfui?</strong></p> <p>Tja – so generell kann man das nicht sagen. Sowohl als (TFS) Benutzer als auch als Software-Entwickler (das läuft in der CLOUD – huiuiuiui) bin ich recht angetan von dem Service, auch wenn ich jetzt direkt keine Verwendung dafür habe. </p> <p>Die Entscheidung den Service bis 5 Mann kostenlos anzubieten ist eine Kampfansage an diverse SVN und Git Hoster – insbesondere da man hier noch ein Build System dazubekommt. Für Entwickler auf der Microsoft Plattform, welche keine eigene Infrastruktur haben oder nicht mehr haben wollen: Einfach mal probieren. <br>Bei größeren Teams oder “ernsthaften” Sachen würde ich mir aber dann doch noch das Pricing anschauen. <br>Bei ganz großen Geschichten (Hallo Konzernmitarbeiter!) muss man vermutlich erstmal mit seinem CTO abklären ob man denn Firmen Source Code in die Hände von Microsoft geben will oder darf ;)</p> <p>Natürlich kann man sich immer streiten ob der <a href="{{BASE_PATH}}/2011/03/20/tfs-segen-oder-fluch/">TFS Segen oder Fluch</a> ist – ist aber ein anderes Thema ;)</p> <p><strong>Ausblick?</strong></p> <p>Es bleibt abzuwarten welche Features Microsoft demnächst “online” schaltet – in <a href="http://blogs.msdn.com/b/bharry/archive/2012/08/28/tfs-shipping-cadence.aspx">diesem Blogpost</a> ist jedenfalls davon die Rede, dass Microsofts Cloud TFS recht kurze Release-Zyklen hat. Bis es den normalen “TFS” einholt dauert es noch etwas. Features wie das Lab-Management stelle ich mir auf einer Cloud-Umgebung ziemlich interessant vor (und hoffe das es keine Unsummen kosten wird). Vermutlich werden einige Sachen auch etwas kniffliger sein als auf der eigenen Umgebung (Debugging des Builds… meist geht das direkt auf dem Build Agent per RDP am besten ;) ). </p> <p><strong>Frage an euch</strong></p> <p>Nutzt jemand bereits aktiv das Angebot oder plant es zu nutzen?</p>
